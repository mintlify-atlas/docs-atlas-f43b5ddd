## What is Biosecurity Assessment?

Biosecurity assessment evaluates the security measures protecting biological materials, data, and facilities from theft, diversion, or misuse. The BioRAM biosecurity assessment provides a structured methodology for identifying vulnerabilities, assessing threat scenarios, and implementing appropriate security countermeasures.

While biosafety focuses on protecting personnel and the environment from accidental exposure, biosecurity addresses intentional threats including unauthorized access, theft of dangerous pathogens, insider threats, and sabotage.

<Note>
The biosecurity assessment is available through the Biosecurity.raml model (Java application) and integrated into the BioRAM-2022 Excel tool. Multiple language versions support international implementation.
</Note>

## Assessment Structure

The biosecurity assessment is organized into comprehensive sections addressing all aspects of security:

### Section 1: Facility and Organizational Context
Establishes the security environment, threat landscape, and institutional security framework.

**Key Elements:**
- Facility location and characteristics
- Organizational security policies
- Regulatory and compliance requirements
- Local threat environment
- Historical security incidents

### Section 2: Material Risk Assessment
Evaluates the attractiveness and consequences of biological material theft or misuse.

**Key Elements:**
- Agent classification and select agent status
- Dual-use research concerns
- Potential misuse scenarios
- Consequence assessment
- Material quantities and accessibility

### Section 3: Security Systems and Controls
Documents existing security measures across all functional areas.

**Key Elements:**
- Physical security infrastructure
- Access control systems
- Inventory management procedures
- Information security controls
- Security personnel and response

### Section 4: Personnel Security Program
Assesses personnel screening, training, and access authorization processes.

**Key Elements:**
- Screening procedures and documentation
- Security training completion
- Access authorization processes
- Insider threat mitigation
- Reporting and accountability culture

### Section 5: Incident Response and Recovery
Evaluates preparedness for security incidents and emergency situations.

**Key Elements:**
- Incident detection capabilities
- Response procedures and teams
- Communication protocols
- Recovery and continuity planning
- Post-incident analysis processes

<Tip>
Engage security professionals, law enforcement liaisons, and institutional security officers early in the assessment process to ensure comprehensive evaluation.
</Tip>

## Risk Scoring Methodology

The biosecurity assessment employs a threat-based risk calculation that considers attractiveness of materials, vulnerability to exploitation, and potential consequences:

### Scoring Components

**Material Attractiveness**
- Select agent status and regulatory classification
- Ease of weaponization or misuse
- Availability from alternative sources
- Technical expertise required for misuse
- Consequence severity of misuse

**Vulnerability Assessment**
- Access control effectiveness
- Inventory accountability rigor
- Personnel security measures
- Detection capabilities
- Response effectiveness

**Consequence Evaluation**
- Public health impact potential
- Economic disruption
- Public confidence and trust
- Regulatory and legal consequences
- Institutional reputation

### Overall Security Risk

The biosecurity risk score integrates these components:

```
Security Risk = (Material Attractiveness × Vulnerability) × Consequence
```

Risk levels are categorized to guide security enhancement priorities:

## Integration with Biosafety

Biosecurity and biosafety programs are complementary and interdependent:

### Shared Elements
- Access control (safety and security)
- Inventory management (accountability and safety)
- Training programs (competency and awareness)
- Incident response (exposures and security events)
- Documentation and record-keeping

### Potential Conflicts
- Emergency egress vs. access control
- Visibility for safety vs. obscurity for security
- Collaboration vs. need-to-know restrictions
- Transparency vs. information protection

<Tip>
**Integrated Approach**

For comprehensive biorisk management, consider using the [Integrated BioRisk Assessment](/assessments/integrated-biorisk) which addresses both safety and security in a unified framework, optimizing measures that support both objectives.
</Tip>

## Best Practices for Security Assessment

### Assessment Preparation
- Involve institutional security officer
- Review previous security assessments
- Gather security system documentation
- Coordinate with IT security staff
- Consider threat intelligence briefings

### Information Handling
- Protect assessment results as sensitive
- Limit distribution to need-to-know personnel
- Store securely with controlled access
- Consider classification for high-risk facilities
- Follow institutional information security policies

### Stakeholder Engagement
- Security professionals and law enforcement
- Facilities and engineering staff
- IT and cybersecurity teams
- Laboratory leadership and staff
- Institutional biosafety and security committees

### Implementation and Follow-up
- Prioritize recommendations by risk reduction
- Develop implementation timeline and budget
- Assign responsibilities for corrective actions
- Monitor implementation progress
- Reassess after security improvements
- Conduct regular security drills and exercises

<Warning>
**Reassessment Triggers**

- Introduction of select agents or high-risk materials
- Security incidents or near-misses
- Facility modifications or relocations
- Significant personnel changes
- Changes in threat environment
- At minimum every 2-3 years
</Warning>
