## Version Overview

BioRAM has evolved through multiple versions since 2016, with separate Excel-based and Java-based implementations. Different assessment types have been updated at different times to reflect evolving biosafety and biosecurity standards.

## Current Versions

### Excel Version

**BioRAM-2022 (BioRisk)** - Latest Integrated Version
- **Status**: Beta (as of December 2023 update)
- **Type**: Combined biosafety and biosecurity assessment
- **Format**: Excel spreadsheet (.xlsx)
- **File Size**: ~204 KB
- **Last Update**: December 2023
- **Use Case**: Comprehensive biorisk assessment in single tool

<Info>
BioRAM-2022 represents the newest approach, integrating both safety and security assessments into a unified workflow. This reduces redundancy and provides a holistic view of laboratory risk.
</Info>

### Java Application

**Version 4.0.5** - Current Java Release
- **Formats**: 
  - `bioram-4.0.5.exe` (Windows executable, ~7.6 MB)
  - `bioram-4.0.5.jar` (Cross-platform JAR, ~7.2 MB)
- **Assessment Models**: Requires separate .raml files for biosafety or biosecurity
- **Languages**: Multiple translations available (English, Spanish, French, Russian, Arabic, Indonesian)
- **Features**: 
  - Save and resume assessments
  - Export results
  - Customizable assessment models
  - Full graphical user interface

## Version Timeline

### 2016 - Initial Releases

**BioRAM-Safety-2016**
- First dedicated biosafety assessment tool
- Focus on laboratory safety risks
- File size: ~148 KB
- Excel-based questionnaire format

**BioRAM-Security-2016**
- Initial biosecurity assessment version
- Focus on intentional misuse and theft risks
- File size: ~267 KB
- Separate from safety assessment

### 2017 - Security Update

**BioRAM-Security-2017**
- Enhanced biosecurity assessment
- Expanded questions and risk factors
- Improved security control evaluation
- File size: ~306 KB (larger question set)
- Maintained separation from safety assessment

<Note>
The 2016 and 2017 versions required users to conduct separate safety and security assessments, which could lead to duplicated effort when both were needed.
</Note>

### 2020 - Security Refinement

**BioRAM-Security-2020**
- Streamlined biosecurity assessment
- Refined question set based on field experience
- Improved usability and clarity
- File size: ~122 KB (more focused)
- Reflects updated biosecurity best practices

### 2022 - Integrated Assessment

**BioRAM-2022 (BioRisk)**
- **Major Innovation**: Combined safety and security in single tool
- **Efficiency**: Eliminates duplicate questions across assessments
- **Comprehensive**: Addresses biorisk holistically
- **Beta Status**: Ongoing refinement based on user feedback
- **2023 Update**: Bug fixes and improvements (December 2023)

### Java Application Evolution

**Version 4.0.5** (Current)
- Enhanced user interface
- Multi-language support (10 language variants)
- Improved file handling and save functionality
- Source code available (Java source.zip)
- Compatible with Windows, macOS, and Linux

**Assessment Model Files (.raml)**
- `Biosafety.raml` - Biosafety assessment model
- `Biosecurity.raml` - Biosecurity assessment model
- Translated versions: Spanish (es), French (fr), Russian (ru), Arabic (AR), Indonesian (id)
- `BiosecurityAugust2015.raml` - Earlier biosecurity model version

## Which Version Should You Use?

### Choose BioRAM-2022 (BioRisk) Excel if:

<Check>
- You need both safety and security assessment
- You prefer working in Excel
- You want the most recent integrated methodology
- You're conducting a comprehensive biorisk evaluation
- You have Microsoft Excel 2007 or later
</Check>

### Choose Separate Excel Versions if:

<Check>
- You only need biosafety OR biosecurity assessment (not both)
- You want the most focused questionnaire for your specific need
- You're using BioRAM-Security-2020 for latest security-only assessment
- You're continuing with previously started assessments
</Check>

### Choose Java Version 4.0.5 if:

<Check>
- You need multi-language support
- You're working on non-Windows platforms without Excel
- You want advanced save/resume functionality
- You need to customize assessment models
- You're conducting multiple assessments and want consistent data management
- You prefer a dedicated application interface
</Check>

## Backward Compatibility

### Excel Versions

- **Not backward compatible**: Each version is a separate file
- **Migration**: No automated migration between versions
- **Recommendation**: Complete assessments in the version you started with
- **New assessments**: Use BioRAM-2022 for comprehensive assessment

### Java Version

- **Model compatibility**: Version 4.0.5 works with older .raml model files
- **Save files**: Assessments saved in v4.0.5 may not open in earlier versions
- **Recommendation**: Use consistent Java version across your organization

<Warning>
If you've started an assessment in an older version, complete it in that version to avoid data loss or inconsistency. Start new assessments in the latest appropriate version.
</Warning>

## Migration Guidance

### Moving to BioRAM-2022

If you're currently using separate safety and security assessments:

1. **Complete existing assessments** in their current versions
2. **Review BioRAM-2022** to understand the integrated approach
3. **Use the Monkey Pox example** to familiarize yourself with the new format
4. **Start new assessments** in BioRAM-2022 for comprehensive evaluation
5. **Retain historical assessments** for comparison and trending

### Adopting Java Version

If switching from Excel to Java:

1. **Install Java Runtime Environment** (JRE 8 or later recommended)
2. **Review BioRAM 4.0 Directions** PDF for setup instructions
3. **Select appropriate .raml file** (Biosafety.raml or Biosecurity.raml)
4. **Test with sample assessment** before production use
5. **Establish file naming conventions** for saved assessments
